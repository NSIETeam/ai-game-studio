**版本**: 1.0

**项目**: AI Game Studio - Demo Game #4

## 📋 游戏概述

### 游戏名称
**Minecraft 2D** (暂定)

### 核心概念
一款基于浏览器的 2D 侧视角像素沙盒游戏，融合《Minecraft》的核心玩法与《Terraria》的视觉风格。玩家可以在无限生成的像素世界中探索、挖掘资源、建造结构，体验经典的沙盒创造乐趣。

### 目标用户
- 休闲游戏玩家
- Minecraft/Terraria 爱好者
- 移动端浏览器用户

### 核心体验
**"随时随地，创造你的世界"**

## 🎯 核心机制

### 1. 移动系统

#### 基础移动
| 操作 | 输入 | 效果 |
|------|------|------|
| 向左移动 | A / ← | 玩家向左行走，速度 4 格/秒 |
| 向右移动 | D / → | 玩家向右行走，速度 4 格/秒 |
| 跳跃 | Space / W / ↑ | 垂直跳跃，高度 3 格 |

#### 物理特性
- **重力加速度**: 20 格/秒²
- **最大下落速度**: 12 格/秒（终端速度）
- **跳跃缓冲**: 地面接触后 0.1 秒内仍可跳跃（手感优化）
- **土狼时间**: 离开平台后 0.1 秒内仍可跳跃
- **空中控制**: 空中水平速度为地面的 70%

#### 碰撞检测
- AABB (轴对齐边界框) 碰撞
- 玩家尺寸: 0.8 × 1.8 格
- 碰撞盒偏移: 水平居中，底部对齐

### 2. 挖掘系统

#### 基本操作
- **鼠标左键**: 破坏方块
- **按住**: 持续挖掘（需要按住 0.3 秒以上）

#### 挖掘机制
| 属性 | 数值 |
|------|------|
| 挖掘范围 | 5 格（以玩家中心计算） |
| 基础挖掘时间 | 0.3 秒/方块 |
| 挖掘动画 | 方块抖动 + 裂纹纹理叠加 |

#### 挖掘规则
- 只能挖掘挖掘范围内的方块
- 挖掘后方块掉落为物品（可拾取）
- 不可挖掘超出范围的方块
- 挖掘进度可视化（裂纹逐渐加深）

### 3. 放置系统

#### 基本操作
- **鼠标右键**: 放置方块
- **放置预览**: 半透明方块预览

#### 放置规则
| 规则 | 说明 |
|------|------|
| 放置范围 | 5 格 |
| 阻挡检测 | 不能与玩家碰撞盒重叠 |
| 支撑检测 | 方块必须有相邻支撑（除特殊方块） |
| 预览显示 | 有效位置绿色，无效位置红色 |

### 4. 背包系统

#### 界面设计
- **位置**: 屏幕底部居中
- **尺寸**: 6 个槽位，每个 40×40 像素
- **选中高亮**: 金色边框 + 放大 1.1 倍

#### 快捷栏操作
| 按键 | 功能 |
|------|------|
| 1-6 | 切换到对应槽位 |
| 鼠标滚轮 | 循环切换槽位 |

#### 物品堆叠
- **最大堆叠**: 64 个/格
- **自动合并**: 相同物品自动堆叠
- **拾取范围**: 2 格半径
- **拾取动画**: 物品飞向玩家

#### 初始物品
玩家开始时拥有：
- 泥土 × 32
- 石头 × 16
- 木头 × 16

## 🧱 方块系统

### 方块属性总览

| ID | 名称 | 颜色 (HEX) | 硬度 | 特殊行为 | 纹理特征 |
|----|------|-----------|------|----------|----------|
| 0 | 空气 | - | - | 透明，可穿透 | - |
| 1 | 泥土 | #8B4513 | 1 | 无 | 深褐色，随机噪点 |
| 2 | 石头 | #808080 | 3 | 无 | 灰色，岩石纹理 |
| 3 | 木头 | #8B5A2B | 2 | 可燃烧 | 木质纹理，年轮 |
| 4 | 树叶 | #228B22 | 0.5 | 透明，可穿透 | 绿色，半透明 |
| 5 | 水 | #1E90FF | - | 流动，可游泳 | 蓝色，动画波动 |
| 6 | 沙子 | #F4A460 | 1 | 受重力影响 | 沙黄色，颗粒感 |

### 方块详细说明

#### 1. 泥土 (Dirt)
- **生成**: 地表 3-5 层
- **特性**: 基础建筑材料
- **纹理**: 16×16 像素，深褐色底色 + 随机深色噪点
- **掉落**: 自身

#### 2. 石头 (Stone)
- **生成**: 泥土下方，地下深处
- **特性**: 较硬，需要更长时间挖掘
- **纹理**: 灰色底色 + 不规则深色斑点模拟岩石
- **掉落**: 自身

#### 3. 木头 (Wood)
- **生成**: 树干部分
- **特性**: 可燃（未来扩展）
- **纹理**: 垂直木质纹理，深浅相间
- **掉落**: 自身

#### 4. 树叶 (Leaves)
- **生成**: 树冠部分
- **特性**: 
  - 半透明（alpha = 0.7）
  - 可穿透（玩家可穿过）
  - 无碰撞体积
- **纹理**: 绿色，叶脉图案
- **掉落**: 自身（低概率）

#### 5. 水 (Water)
- **生成**: 地表水体、地下湖
- **特性**:
  - 流动动画（8 帧循环，0.5 秒/循环）
  - 玩家可在水中游泳（速度减半）
  - 无碰撞体积
  - 不可挖掘，不可放置（自然生成）
- **纹理**: 蓝色渐变 + 波浪动画

#### 6. 沙子 (Sand)
- **生成**: 沙漠区域、海滩
- **特性**:
  - **重力影响**: 下方无支撑时下落
  - 下落速度: 8 格/秒
  - 落地时产生轻微粒子效果
- **纹理**: 沙黄色 + 细颗粒噪点
- **掉落**: 自身

### 方块渲染

#### 基础渲染
```
每个方块: 16×16 像素纹理
渲染尺寸: 根据缩放级别动态调整
基础尺寸: 32×32 屏幕像素（1x 缩放）
```

#### 光影效果
- **高度渐变**: 根据 Y 坐标调整亮度
  - 地表 (Y=32): 100% 亮度
  - 地下每深 10 格: -5% 亮度
  - 最低亮度: 30%
- **方块边缘**: 1 像素深色描边增强像素感

## 🌍 地形生成算法

### 世界规格
| 属性 | 数值 |
|------|------|
| 世界宽度 | 512 格（可扩展为无限） |
| 世界高度 | 128 格 |
| 地表高度 | 32 格（基准） |
| 区块大小 | 16×16 格 |

### 生成流程

#### 1. 基础地形生成
```javascript
// 伪代码
for each x in worldWidth:
    // 使用 Simplex 噪声生成高度
    heightNoise = simplex(x * 0.05) * 8
    surfaceY = 32 + heightNoise
    
    for y from 0 to surfaceY:
        if y < surfaceY - 3:
            block = STONE
        else:
            block = DIRT
```

#### 2. 洞穴生成（简化版）
- 使用 2D Simplex 噪声
- 阈值 < 0.3 时生成空气（洞穴）
- 仅在地表下 5 格以下生成

#### 3. 水体生成
- 识别低洼地形（高度 < 28）
- 填充水方块至高度 28
- 水边缘平滑过渡

#### 4. 树木生成
```javascript
// 生成条件
if random() < 0.05 and block[x][surfaceY] == DIRT:
    // 生成树干
    for h in 0 to 4:
        block[x][surfaceY - h] = WOOD
    
    // 生成树叶
    for dx in -2 to 2:
        for dy in -2 to 0:
            if distance(dx, dy) <= 2.5:
                block[x+dx][surfaceY-4+dy] = LEAVES
```

#### 5. 沙地生成
- 地表高度 < 25 的区域
- 替换表层泥土为沙子
- 生成沙漠地形特征

### 噪声参数
| 噪声类型 | 频率 | 振幅 | 用途 |
|----------|------|------|------|
| 地形高度 | 0.05 | 8 | 基础起伏 |
| 细节地形 | 0.1 | 2 | 表面细节 |
| 洞穴 | 0.08 | 1 | 地下空洞 |

### Simplex 噪声实现（简化版）
由于文件大小限制，使用简化版噪声生成器：
```javascript
// 基于位运算的快速伪随机
function noise(x) {
    let n = Math.sin(x * 12.9898) * 43758.5453;
    return n - Math.floor(n);
}

// 平滑插值
function smoothNoise(x) {
    return noise(x)/2 + noise(x-1)/4 + noise(x+1)/4;
}
```

// 二维数组存储方块 ID
world = Uint8Array[worldWidth * worldHeight]

// 访问方式
function getBlock(x, y) {
    if (x < 0 || x >= worldWidth || y < 0 || y >= worldHeight) 
        return BLOCK_STONE; // 边界外默认石头
    return world[y * worldWidth + x];
}

### 渲染策略

#### 视口管理
```javascript
viewport = {
    x: player.x - canvas.width / 2 / tileSize,
    y: player.y - canvas.height / 2 / tileSize,
    width: canvas.width / tileSize + 2,  // +2 用于边缘缓冲
    height: canvas.height / tileSize + 2
};
```

#### 渲染流程
1. **清空画布**: `clearRect`
2. **绘制背景**: 天空渐变（根据时间）
3. **绘制方块**: 仅渲染视口内方块
4. **绘制物品**: 掉落物
5. **绘制玩家**: 简单像素角色
6. **绘制 UI**: 快捷栏、准星

#### 性能优化
- **脏矩形**: 仅重绘变化区域
- **视口裁剪**: 不渲染屏幕外方块
- **区块缓存**: 静态地形可缓存为离屏 Canvas

### 游戏循环
```javascript
function gameLoop() {
    const now = performance.now();
    const dt = (now - lastTime) / 1000;
    lastTime = now;
    
    update(dt);      // 更新游戏逻辑
    render();        // 渲染画面
    
    requestAnimationFrame(gameLoop);
}
```

### 物理更新
```javascript
function updatePhysics(dt) {
    // 重力
    player.vy += gravity * dt;
    
    // 速度限制
    player.vy = Math.min(player.vy, maxFallSpeed);
    
    // 位置更新
    player.x += player.vx * dt;
    player.y += player.vy * dt;
    
    // 碰撞检测与响应
    resolveCollisions();
}
```

## 📱 移动端适配

### 响应式设计
- **最小支持**: 320px 宽度
- **推荐尺寸**: 480px 以上
- **自适应**: UI 元素根据屏幕尺寸缩放

### 性能考虑
- **目标帧率**: 30 FPS（移动端）
- **渲染优化**: 降低渲染距离
- **物理简化**: 减少每帧物理迭代次数

### 触摸优化
- **按钮尺寸**: 最小 60×60 像素（触控友好）
- **响应延迟**: < 50ms
- **误触防护**: 长按确认重要操作
